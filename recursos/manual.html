1. Introducción

Transcriptor Médico Pro es una aplicación web profesional que convierte dictados médicos en informes estructurados de alta calidad, listos para imprimir o archivar.

¿Qué hace la app?

  1. Graba dictados médicos en tiempo real o acepta archivos de audio preexistentes
  2. Transcribe automáticamente usando inteligencia artificial (Whisper de Groq)
  3. Detecta el tipo de estudio médico entre más de 50 especialidades y estudios
  4. Estructura el informe con plantillas profesionales (LLaMA 3.3-70b vía Groq)
  5. Exporta como PDF profesional con encabezado, firma digital, QR y datos completos del paciente
  6. Guarda historial de informes, registro de pacientes, médicos solicitantes y motivos de estudio

Características principales

  • +50 plantillas médicas especializadas en 15 categorías (Neumología, Cardiología, Neurología, Oftalmología, Imágenes, etc.)
  • Detección automática del tipo de estudio a partir del texto dictado
  • PDF profesional con logo institucional, logo profesional, firma digital y código QR de verificación
  • Diccionario médico con corrección automática de términos y revisión por IA
  • Registro de pacientes, médicos solicitantes y motivos con autocompletado inteligente
  • Historial completo de informes con búsqueda y restauración
  • Editor rich-text con formato completo, tablas, imágenes, formas y búsqueda/reemplazo
  • Versionado automático del editor (snapshots) con comparación lado a lado
  • 5 formatos de exportación: PDF, RTF (Word), HTML, TXT y email directo
  • Modo multi-pestaña: trabaje con múltiples informes simultáneamente
  • Soporte multi-profesional para clínicas (PLAN CLINIC)
  • Modo oscuro / claro con personalización de colores y skins
  • Funciona offline para edición y exportación (PWA instalable)
  • Telemetría automática de errores para diagnóstico y soporte remoto

3. Planes y licencias

La app ofrece distintos planes según las necesidades del profesional:

Cargando tabla de planes…

Diferencias clave entre NORMAL y PRO

  • Modo PRO: En modo PRO, al transcribir el audio la app detecta automáticamente el tipo de estudio y estructura el informe con IA. En modo NORMAL se transcribe y luego se selecciona manualmente la plantilla.
  • Formatos de descarga: NORMAL solo exporta PDF y TXT. PRO agrega RTF (Word), HTML y envío por email.
  • Historial: PRO almacena más informes y permite restaurar cualquiera.
  • Revisión IA inline: Solo PRO permite la revisión párrafo por párrafo con sugerencias de IA.
  • Dictado por campo: Solo PRO permite grabar audio directamente dentro de cada campo del informe estructurado.

Plan CLINIC

Diseñado para instituciones con múltiples profesionales. Incluye:

  • Panel de administración de profesionales (agregar, editar, desactivar)
  • Login por profesional con PIN de 4 dígitos
  • Cada profesional tiene datos, firma y configuración independiente
  • Selector rápido de profesional activo en el encabezado
  • Múltiples lugares de trabajo configurables
  • Las plantillas se filtran automáticamente según las especialidades del profesional activo

Límite de dispositivos

Cada plan tiene un límite de dispositivos simultáneos. Si alcanzás el límite, la app mostrará una opción para contactar soporte o liberar dispositivos.

💡 Un "dispositivo" corresponde a un navegador en una computadora o celular. Usar Chrome y Edge en la misma PC consume 2 dispositivos.

4. Instalación y acceso

4.1 Acceso web

Abrí la URL proporcionada por tu administrador en un navegador compatible. No requiere instalación.

4.2 Instalar como app (PWA)

En Chrome o Edge (escritorio):

  1. Abrí la app en el navegador
  2. Si aparece el botón 📲 Instalar app en el encabezado, hacé clic
  3. Confirmá la instalación en el diálogo del navegador
  4. La app aparece en tu escritorio como aplicación independiente

En Android (Chrome):

  1. Abrí la URL en Chrome
  2. Tocá el menú ⋮ → "Instalar aplicación"
  3. Confirmá

En iOS (Safari):

  1. Abrí la URL en Safari
  2. Tocá Compartir (↑)
  3. Seleccioná "Agregar a pantalla de inicio"
  4. Confirmá

4.3 Ventajas de instalar como PWA

  • Se abre en ventana propia sin barra del navegador
  • Acceso rápido desde escritorio o pantalla de inicio
  • Funciona offline para edición, configuración y exportación
  • Actualizaciones automáticas transparentes
  • Menor consumo de memoria

6. Configuración de la API Key

La transcripción y estructuración con IA requieren una API Key de Groq.

6.1 Obtener la API Key

  1. Ingresá a console.groq.com/keys
  2. Creá una cuenta gratuita si aún no tenés
  3. Hacé clic en "Create API Key"
  4. Copiá la key generada

6.2 Ingresar la key en la app

  • En el onboarding (paso 2): pegá la key en el campo correspondiente
  • O en cualquier momento: ⚙️ Configuración → API Key de Groq
  • Pegá la key y hacé clic en 💾 Guardar
  • Hacé clic en 🔌 Probar para verificar que funciona

6.3 Indicador de estado

IndicadorSignificado
🟢 ConectadaLa key es válida y la API responde correctamente
🔴 No configuradaFalta la key o la key es inválida

6.4 Keys de respaldo

El sistema soporta hasta 2 keys de respaldo (b1, b2). Si la key principal falla, la app intentará automáticamente con las respaldo antes de mostrar error.

7. Flujo principal de trabajo

El flujo estándar para generar un informe médico es:

  1. Capturar audio — Grabar en vivo o subir archivos de audio
  2. Transcribir — La IA convierte el audio en texto
  3. Estructurar — La IA organiza el texto en secciones según la plantilla médica correspondiente
  4. Revisar y editar — Ajustar campos, corregir, completar datos faltantes
  5. Cargar datos del paciente — Nombre, DNI, médico solicitante, etc.
  6. Exportar — Descargar como PDF, RTF, HTML o TXT; o enviar por email

8. Grabación y carga de audio

8.1 Grabar en vivo

  1. Hacé clic en el botón 🎙️ Grabar en el panel lateral
  2. Permitir el acceso al micrófono cuando el navegador lo solicite
  3. Dictá el informe. Un contador de tiempo muestra la duración
  4. Hacé clic en ⏹ Detener para finalizar
  5. El audio queda listo para transcribir

8.2 Subir archivos de audio

  • Arrastrá uno o varios archivos a la zona de arrastre, o hacé clic para seleccionar
  • Formatos aceptados: MP3, WAV, OGG, M4A
  • Tamaño máximo por archivo: 25 MB
  • Podés subir múltiples archivos simultáneamente

8.3 Opciones de procesamiento

OpciónDescripción

🪄 Reducir ruidoFiltra ruido de fondo para mejorar la claridad

8.4 Grabar adicional (post-transcripción)

Después de transcribir, podés agregar más audio con el botón 🎙️+ en la barra del editor. El nuevo texto se agrega al final del existente.

9. Transcripción

Una vez capturado el audio:

  1. Hacé clic en 📥 Transcribir
  2. La barra de progreso muestra el avance
  3. El texto transcrito aparece en el editor

Motor de transcripción

La app utiliza Whisper (whisper-large-v3-turbo) de Groq, un modelo de IA especializado en transcripción de audio a texto con soporte para español médico.

Errores y reintentos

Si la transcripción falla:

  • La app intenta automáticamente con keys de respaldo (si las hay)
  • Muestra un botón 🔄 Reintentar archivos fallidos para los segmentos que fallaron
  • Errores comunes: key inválida (401), límite de tasa (429), sin conexión (network)

Modo PRO — Fuente de texto

En modo PRO, además de audio, podés pegar texto directamente o subir archivos de texto:

  • Formatos aceptados: TXT, DOC, DOCX, PDF
  • Útil para estructurar informes viejos, escaneados o de otro sistema
  • Hacé clic en "Estructurar texto" para procesar directamente

Autodetección de plantilla (Modo PRO)

En Modo PRO, la app analiza el texto transcrito y detecta automáticamente qué plantilla corresponde usando un sistema de palabras clave médicas. Por ejemplo:

  • "espirometría CVF VEF1" → detecta Espirometría
  • "holter ritmo sinusal extrasístoles" → detecta Holter ECG
  • "protocolo Bruce METs" → detecta Ergometría

Selección manual de plantilla

Podés elegir o cambiar la plantilla en cualquier momento:

  • Modo Normal: El dropdown "📋 Plantilla" en la barra del editor permite seleccionar
  • Modo PRO: El botón 🔄 Cambiar plantilla aparece después de estructurar
  • Wizard lateral: El panel "Asistente de Plantilla" permite aplicar estructura con IA en cualquier momento

Filtrado por especialidad

Las plantillas disponibles se filtran según:

  • El plan del usuario (NORMAL ve un subconjunto, PRO ve todas)
  • En CLINIC: las especialidades del profesional activo determinan qué plantillas aparecen

11. Estructuración con IA

La estructuración toma el texto libre dictado y lo organiza en secciones médicas formales.

¿Cómo funciona?

  1. La IA recibe el texto transcrito + el prompt de la plantilla seleccionada
  2. Organiza la información en secciones estándar (ej: HALLAZGOS, MEDICIONES, CONCLUSIÓN)
  3. Cada campo se convierte en un span editable clickeable en el editor
  4. Los campos no mencionados en el dictado se marcan como [No especificado] o s/p

Reglas de la IA

  • NUNCA inventa datos que no fueron dictados
  • Si una sección no fue mencionada, la marca como [No especificado]
  • La conclusión solo incluye hallazgos positivos/patológicos dictados
  • Mantiene terminología médica formal en español
  • Respeta el formato estándar de cada tipo de estudio

Comparación Original vs Estructurado

Después de estructurar, podés comparar ambas versiones:

  • ↩ Original: Muestra el texto antes de estructurar
  • ⧉ Comparar: Vista lado a lado simultánea

  • Campo de reemplazo: reemplazar uno o todos

12.4 Edición de campos estructurados

Después de estructurar, cada campo del informe es clickeable. Al hacer clic se abre un modal de edición con:

  • Tab Escribir: textarea con el contenido actual + 3 botones rápidos:
    • s/p — Sin particularidades
    • Sin particularidades — Texto completo
    • No evaluado
    Además, chips dinámicos con sugerencias contextuales según el tipo de campo.
  • Tab Grabar (solo PRO): Micrófono para dictar directamente el contenido del campo
  • Dejar en blanco: Vacía el campo
  • Eliminar sección: Remueve el campo completo del informe

12.5 Multi-pestaña

Podés trabajar con múltiples informes a la vez. Cada transcripción o informe abierto se muestra como una pestaña en la barra superior del editor.

12.6 Snapshots y versionado

La app captura automáticamente versiones del informe en puntos clave (post-transcripción, post-estructuración). Podés navegar entre versiones anteriores y restaurar cualquiera.

12.7 Autoguardado

El editor guarda automáticamente cada 30 segundos. Si cerrás el navegador accidentalmente, al reabrir aparece el botón ♻️ Restaurar sesión para recuperar el trabajo.

13. Revisión IA inline (PRO)

La revisión IA inline analiza el informe y propone correcciones párrafo por párrafo.

¿Cómo usarla?

  1. Activá el toggle ▶ Rev IA en la barra del editor
  2. La IA revisa el texto actual y muestra sugerencias inline
  3. Cada sugerencia se puede aceptar o rechazar individualmente

¿Qué revisa?

  • Terminología médica incorrecta o inconsistente
  • Formato y estructura del informe
  • Omisiones evidentes en la conclusión
  • Concordancia entre hallazgos y conclusión

14. Chequeo médico

El botón 🩺 Medical Check en la barra del editor ejecuta una verificación rápida del informe:

  • Detecta campos vacíos o con [No especificado]
  • Lista las secciones que podrían necesitar revisión
  • Muestra un resumen de completitud del informe

15. Diccionario médico

El diccionario médico permite definir reemplazos automáticos de términos en los informes.

15.1 Acceso

⚙️ Configuración → Herramientas → 📖 Diccionario médico, o desde la barra del editor.

15.2 Pestañas del diccionario

  • 📋 Revisión: Lista de correcciones encontradas en el texto actual. Checkbox para aplicar cada una. Botón "Buscar con IA" para sugerencias adicionales.
  • 📖 Diccionario: Tabla de entradas personalizadas. Cada entrada tiene un término original y su reemplazo. Podés agregar, editar y eliminar entradas.

15.3 Funcionamiento

Al transcribir o estructurar, el diccionario aplica automáticamente los reemplazos definidos. Por ejemplo:

  • vef1 → VEF1
  • dlco → DLCO
  • presion arterial → presión arterial

17. Registro de pacientes

La app mantiene un registro local de todos los pacientes atendidos.

Acceso

⚙️ Configuración → Herramientas → 👥 Registro de pacientes

Funcionalidades

  • Búsqueda: Filtrar por nombre, DNI o cualquier campo
  • Edición inline: Hacer clic en cualquier campo para editarlo
  • Agregar: Los pacientes se agregan automáticamente al generar un informe, o manualmente
  • Eliminar: Con confirmación
  • Exportar: Descargar como JSON o CSV
  • Importar: Cargar desde archivo JSON

Datos almacenados por paciente

Nombre, DNI, edad, sexo, obra social, nº afiliado, fecha de última visita, cantidad de visitas.

Funcionalidades

  • CRUD completo (agregar, editar, eliminar)

20. Historial de informes

La app guarda automáticamente cada informe generado.

Acceso

⚙️ Configuración → Herramientas → 📋 Historial de informes

Funcionalidades

  • Búsqueda por paciente, tipo de estudio o fecha
  • Restaurar: Carga un informe previo en el editor para re-editarlo o re-imprimirlo
  • Exportar: Descargar como JSON o CSV
  • Importar: Cargar desde JSON
  • Eliminar: Individual o limpiar todo (con confirmación)

Datos de cada registro

Nombre del paciente, tipo de estudio/plantilla, fecha, hora, contenido completo del informe, número de informe asignado.

Límite de almacenamiento

El límite depende del plan: TRIAL almacena los últimos 10, NORMAL 20, PRO 30, CLINIC 100. Al exceder el límite, los más antiguos se descartan automáticamente.

Cada pestaña tiene su propia barra de búsqueda, tabla de datos y acciones CRUD (crear, editar, borrar).

23. Vista previa e impresión

Vista previa

El botón 👁️ Vista previa abre un modal con la representación exacta del PDF en formato A4, incluyendo encabezado, datos del paciente, cuerpo del informe y pie con firma.

Acciones disponibles

  • 📥 Descargar: Descarga el PDF (o el formato favorito si se configuró otro)
  • 🖨️ Imprimir: Abre el diálogo de impresión del navegador
  • 📧 Enviar por email: Abre un modal para enviar el informe como adjunto

Número de informe

Si la opción "Número de informe" está activada, cada informe recibe un código único con el formato: TIPO-NNNNN-DDMMAA (ej: ESPIRO-00015-260326). El contador es auto-incremental por tipo de estudio.

45. Telemetría de errores

La app captura automáticamente errores JavaScript no manejados y los envía al backend para análisis.

¿Qué se captura?

  • Errores JS no capturados (window.onerror)
  • Promesas rechazadas sin manejo (unhandledrejection)
  • Mensaje de error, archivo fuente, línea, stack trace
  • Versión de la app y URL donde ocurrió

Protecciones

  • Los errores se agrupan en lotes (batch) y se envían cada 60 segundos
  • Se deduplican: errores idénticos dentro de 5 segundos se reportan una sola vez
  • Máximo 50 errores por sesión para evitar saturar el backend
  • Al cerrar la app, los errores pendientes se envían con sendBeacon
  • No se captura información personal del paciente

46. Seguridad y privacidad

Datos locales

  • Todos los datos del paciente, informes, registro y configuración se almacenan localmente en el navegador (localStorage)
  • La API Key se guarda en el navegador y se envía directamente a Groq, nunca a terceros
  • El audio se envía a Groq para transcripción pero no se almacena en sus servidores

Datos en el backend

  • El backend (Google Apps Script + Sheets) almacena: datos de licencia, plan, dispositivos registrados, métricas de uso, mensajes de soporte
  • NO almacena informes médicos, datos de pacientes ni audios

Protecciones implementadas

  • Sanitización contra XSS en todas las entradas del usuario
  • Device ID generado con crypto.randomUUID()
  • Validación de fecha de vencimiento en caché offline
  • Límite de reintentos en contacto (máx 10)
  • Límite de registros de historial por plan

48. Gestión de almacenamiento

La app usa localStorage del navegador. Cada navegador tiene un límite (~5–10 MB).

¿Qué consume espacio?

  • Historial de informes: El contenido completo de cada informe
  • Logos y firmas: Imágenes codificadas en base64
  • Registro de pacientes: Datos de todos los pacientes
  • Configuración: Preferencias, perfiles, diccionario

Si el almacenamiento se llena

La app muestra una notificación y elimina automáticamente los informes más antiguos del historial. Para liberar espacio manualmente:

  • Exportar y luego limpiar el historial de informes
  • Eliminar logos/firmas que ya no uses
  • Reducir el tamaño de las imágenes antes de subirlas

Podés ver el espacio usado en ⚙️ Configuración → Info y soporte.

49. Resetear la app

  • Requiere una contraseña de administrador

  • No afecta al backend (los datos de licencia y plan se mantienen)

50. Solución de problemas

La transcripción falla

  • Verificá que la API Key sea válida (⚙️ → API Key → 🔌 Probar)
  • Verificá la conexión a internet
  • Si el error es 429 (límite de tasa), esperá unos segundos e intentá de nuevo
  • Si el audio es muy largo (~25 MB), dividilo en partes más cortas

El micrófono no funciona

  • Verificá que el navegador tenga permiso para acceder al micrófono
  • Probá en Chrome (mejor soporte de MediaRecorder)
  • Cerrá otras apps que puedan estar usando el micrófono

El PDF no se genera correctamente

  • Verificá que los datos del profesional estén completos
  • Si el logo se ve pixelado, usá una imagen de mayor resolución
  • Para firmas, usá PNG con fondo transparente

La app se ve desactualizada

  • Andá a ⚙️ → Actualizar App → Forzar actualización
  • O limpiá la caché del navegador (Ctrl+Shift+Del)

"Límite de dispositivos alcanzado"

  • Tu plan permite un número máximo de dispositivos (navegadores)
  • Contactá al administrador para liberar dispositivos o upgradear tu plan
  • Usá el botón 📧 Contacto → Motivo: "Límite de dispositivos alcanzado"

"Licencia expirada"

  • Tu período de prueba (TRIAL) o suscripción venció
  • Contactá al administrador o usá el portal de pagos para renovar
